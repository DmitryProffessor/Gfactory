# GFactory: Multimodal RAG для сложных промышленных документов

## Обзор проекта
GFactory — это программное решение для автоматизированного поиска и анализа информации в неструктурированных промышленных документах (технологические регламенты, отчеты, схемы). 

Основная проблема современных RAG-систем — потеря контекста при работе со сложной версткой и графикой (таблицы, чертежи, диаграммы). Наш проект решает это через **OCR-Free** подход, используя визуальные эмбеддинги (ColPali) и локальные Vision-LLM (Moondream2), что позволяет системе «видеть» документ целиком, а не только извлеченный текст.

## Ключевые технические особенности

### 1. Гибридная балансировка ресурсов (GPU + CPU)
Для обеспечения работы на потребительском железе (уровня RTX 3060 12GB) реализована схема разделения нагрузки:
*   **GPU:** Задействован для высокоскоростного поиска по визуальному индексу с использованием модели **ColPali v1.2** (через библиотеку Byaldi). Это обеспечивает миллисекундный отклик при поиске по тысячам страниц.
*   **CPU + RAM:** Используются для инференса Vision-LLM **Moondream2**. Такое разделение предотвращает ошибки Out of Memory (OOM) и позволяет запускать систему On-premise без необходимости в серверных картах класса A100.

### 2. Высокопроизводительный Pipeline индексации
Процесс подготовки данных оптимизирован для работы в реальном времени:
*   **Parallel Rendering:** С использованием `ThreadPoolExecutor` и библиотеки `PyMuPDF` (fitz) нарезка PDF на изображения происходит в многопоточном режиме, что ускоряет процесс в 2-4 раза.
*   **Fast Mode (Dynamic DPI):** Возможность переключения разрешения рендеринга. Низкое DPI для быстрой предварительной индексации и высокое — для финальной обработки.

### 3. OCR-Free Reasoning (Интеллектуальный анализ)
В отличие от классических систем, которые передают в LLM только текст, наш пайплайн:
1. Находит наиболее релевантную страницу по визуальным признакам.
2. Передает изображение страницы в локальную модель Moondream2.
3. Модель анализирует визуальный контекст (например, схему «закрепления аполярного собирателя») и дает ответ на основе того, что реально изображено на чертеже.

### 4. Полная приватность и ИБ
Система работает полностью автономно (Air-gapped ready). Все компоненты (Python, PyTorch, Byaldi, Moondream) загружаются локально, исключая передачу данных во внешние API (OpenAI, Anthropic и др.), что является критическим требованием для индустриальных компаний.

## Структура проекта

```text
GFACTORY/
├── data/                   # Данные для обработки
│   ├── raw/                # Исходные PDF документы
│   ├── processed_images/   # Результаты обработки (нарезка страниц)
│   └── temp_pdf_pages/     # Временные файлы для UI-демо
├── notebooks/              # Эксперименты и прототипы (demo.ipynb)
├── scripts/                # Утилиты и демо-скрипты
│   ├── demo_search.py      # Быстрый поиск по индексу (CLI)
│   ├── local_multimodal_rag.py # Полный цикл: поиск + анализ на CPU
│   └── run_query.py        # Скрипт для одиночных запросов
├── src/                    # Модульный исходный код системы
│   ├── chunker.py          # Логика разбиения документов
│   ├── indexer.py          # Логика создания визуальных индексов
│   ├── retriever.py        # Поисковый движок
│   ├── generator.py        # Интерфейс работы с Vision-LLM
│   └── pipeline.py         # Объединяющий класс системы
├── app.py                  # Графический интерфейс (Gradio)
├── requirements.txt        # Список зависимостей
└── setup.py                # Скрипт установки проекта как пакета
```

## Установка и запуск

### Предварительные требования
*   Python 3.10+
*   NVIDIA GPU (рекомендуется 12GB+ VRAM)
*   CUDA Toolkit

### Установка
1. Клонируйте репозиторий.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

### Запуск интерфейса
Для запуска интерактивного демо с возможностью загрузки PDF и чата:
```bash
python app.py
```
*Интерфейс будет доступен по адресу `http://localhost:7860`.*

### Запуск CLI-демо
Для проверки работы полнотекстового визуального поиска:
```bash
python scripts/local_multimodal_rag.py
```

## Реализованные компоненты (согласно задачам трека)

1.  **Автоматизация индексации:** Интегрирован механизм автоматического превращения PDF-архивов в векторный визуальный индекс.
2.  **Поиск по сложным запросам:** Система успешно обрабатывает запросы к таблицам и схемам благодаря ColPali.
3.  **Аналитика:** Moondream2 позволяет извлекать конкретные данные (например, "Долевое участие металлов в EBITDA") напрямую из визуального представления отчетов.
4.  **Масштабируемость:** Модульная структура в папке `src/` позволяет легко заменить поисковый движок или модель генерации без переписывания всего проекта.

## Технические оптимизации для хакатона
*   **Image Scaling:** Перед передачей в Vision-LLM изображение масштабируется до 768x768 (thumbnail), что сокращает время инференса на CPU на 30% без потери качества распознавания ключевых элементов.
*   **Memory Management:** Принудительная очистка кэша GPU (`torch.cuda.empty_cache()`) и сборщик мусора (`gc.collect()`) вызываются после каждого этапа индексации и поиска для стабильной работы приложения в течение длительного времени.
