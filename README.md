# Gfactory: Multimodal Vision RAG 🚀
**Инструмент для поиска и анализа технологической документации на основе зрения и языка**

Gfactory — это интеллектуальная система, которая преобразует сложные технические документы (PDF, схемы, таблицы) в структурированные ответы. В отличие от стандартных систем, наше решение "видит" документы целиком, позволяя находить и интерпретировать графическую информацию — чертежи флотационных машин, химические схемы и графики — без использования потерь при конвертации текста (OCR).

# Технологический стек Gfactory

Система объединяет лучшие практики текстового RAG и инновационные подходы визуального анализа (Vision-Language Models).

---

## 👁 Мультимодальный поиск и индексация (Retrieval)

### [ColPali](https://huggingface.co/vidore/colpali-v1.2) & [Byaldi](https://github.com/answerdotai/byaldi)
Центральный компонент нашего решения для трека Мультимодальных RAG.
- **ColPali (Vision-Language Retrieval)** — модель, которая индексирует страницы документов как изображения. Она создает визуальные эмбеддинги, позволяя системе находить схемы по их структуре и контексту, а не только по словам.
- **Byaldi** — высокоуровневая библиотека для управления индексом ColPali. Она реализует механизм **Late Interaction**, обеспечивая точность поиска (Score) на уровне лучших текстовых моделей при работе с картинками.

### [FAISS](https://github.com/facebookresearch/faiss)
Используется для сверхбыстрого векторного поиска. Позволяет за миллисекунды находить наиболее релевантные страницы среди тысяч проиндексированных документов.

## 🧠 Интерпретация и генерация (Interpretation)

### [Moondream2](https://moondream.ai/) (Local Vision LLM)
Миниатюрная, но мощная Vision-модель (1.6B параметров), интегрированная для анализа найденных изображений.
- **Полная автономность:** Работает локально, обеспечивая безопасность данных (On-premise).
- **Технический анализ:** Способна описывать элементы схем (воздух, вода, реагенты) и объяснять их взаимодействие.

### [Zephyr-7B Beta](https://huggingface.co/HuggingFaceH4/zephyr-7b-beta) (Text Reader)
Используется в текстовом пайплайне для синтеза ответов на основе извлеченного текста и метаданных.

## ⚙️ Инфраструктура и оптимизация

### [PyTorch 2.6](https://pytorch.org/) & [CUDA 12.4](https://developer.nvidia.com/cuda-toolkit)
Основа вычислений, оптимизированная под современный стек (Python 3.13) и видеокарты **NVIDIA RTX 3060**. Мы реализовали гибридный инференс: быстрый поиск на GPU и стабильную генерацию на CPU/GPU.

### [PyMuPDF](https://pymupdf.readthedocs.io/)
Высокопроизводительная библиотека для рендеринга PDF в изображения высокого разрешения (DPI), что критично для качественной работы Vision-моделей без зависимости от громоздких системных утилит (Poppler).

## ✨ Ключевые возможности (Features)

| Функция | Описание |
|---------|-------------|
| 🖼 **Vision-First Indexing** | Индексация страниц как изображений. Схемы и таблицы сохраняют 100% контекста. |
| 🔍 **Semantic Visual Search** | Поиск по визуальным признакам: "найди схему флотации" — система вернет чертеж, а не страницу с упоминанием слова. |
| 🛡 **On-premise Security** | Весь пайплайн работает локально. Ваши чертежи и отчеты не покидают контур безопасности. |
| ⚡️ **Sub-second Retrieval** | Время поиска по базе документов составляет менее 0.5 сек. |
| 📊 **Complex Table Support** | Корректная обработка таблиц любой сложности за счет визуального считывания структуры. |
